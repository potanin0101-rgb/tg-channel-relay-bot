# Telegram Channel Relay Bot

Бот берёт новые посты из `source`-канала и публикует их в один или несколько `target`-каналов в лайв-режиме.

Для каждого маршрута `source -> target` можно задать свою замену текста:

- менять телефон
- менять ссылку
- менять любой фрагмент текста
- пропускать только нужные марки, модели или темы

## Что умеет

- один source -> много target-каналов
- отдельная замена текста на каждом маршруте
- include / exclude фильтры по ключевым словам
- включение и выключение маршрутов
- кнопочное меню в личке бота
- работа в контексте конкретного promo-канала
- повторная выгрузка истории только в выбранный promo-канал
- текстовые посты
- одиночные медиа
- альбомы
- JSON-хранилище без базы данных

## Ограничения

- бот должен быть администратором во всех каналах
- после замены текста Telegram-форматирование может не сохраниться, если длина текста изменилась
- кнопки под исходными постами не копируются
- экзотические типы сообщений могут уйти через `copyMessage` только без замены текста

## Настройка

### 1. Создайте бота

Через `@BotFather` получите токен и вставьте его в `.env.local`.

Бот понимает оба имени переменной:

- `BOT_TOKEN`
- `TELEGRAM_BOT_TOKEN`

### 2. Добавьте бота в каналы

Во все исходные и целевые каналы бот должен быть добавлен как администратор.

### 3. Подключите каналы в личке с ботом

Теперь основной сценарий такой:

1. Откройте `/start`
2. Нажмите `Подключить`
3. Выберите `Добавить source` или `Добавить promo`
4. Для публичного канала отправьте в личку боту:

```text
@mainchannel main
```

или

```text
@promo1channel promo-1
```

5. Для приватного канала можно:

- сначала отправить alias, например `promo-1`
- потом переслать в личку боту любой пост из этого канала

### 4. Создайте маршруты

Теперь маршрут лучше создавать кнопками:

1. Откройте `Промо-каналы`
2. Выберите нужный promo-канал
3. Нажмите `Добавить source в этот канал`
4. Выберите source-кнопку

После этого откроется экран именно этого маршрута.

### 5. Задайте замены

Из экрана маршрута:

- `Замена текста` — пришлите правило `старый => новый`
- `Контактный блок` — пришлите целиком новый хвост поста

Так настройки будут применяться только к выбранному promo-каналу.

### 5a. Замените весь контактный хвост на свой блок

Нажмите в нужном маршруте кнопку `Контактный блок` и пришлите новый блок одним сообщением.

Бот заменит все, начиная с заданного маркера контактов, только в этом маршруте.

### 6. Задайте фильтры по бренду или теме

Если есть отдельный канал только под BMW, откройте его маршрут и нажмите `Include`, затем пришлите:

```text
bmw,x5,x6,m3,m5
```

Если нужно отсечь конкурентов, нажмите `Exclude` и пришлите:

```text
audi,mercedes,toyota
```

Теперь в `bmw-only` попадут только посты, где в тексте или подписи есть слова из include-списка и нет слов из exclude-списка.

### 7. Повторно выгрузите последние посты

Новый основной сценарий:

1. Откройте нужный promo-канал
2. Откройте нужный маршрут
3. Нажмите `История`
4. Выберите `10`, `20` или `50` постов

В таком режиме история уходит только в этот выбранный promo-канал, а не во все маршруты source.

## Меню

В личке отправьте:

```text
/start
```

Меню покажет:

- статус
- promo-каналы
- source-каналы
- подключение новых каналов
- помощь

Внутри выбранного promo-канала можно:

- открыть нужный маршрут
- включать и выключать его
- настраивать контактный блок
- задавать include / exclude фильтры
- повторно выгружать 10 / 20 / 50 последних постов только в него

## Команды в личке

```text
/start
/menu
/cancel
/help
/status
/source_add @channel_username alias
/target_add @channel_username alias
/target_add alias
/route_add source_alias target_alias
/route_remove source_alias target_alias
/route_replace source_alias target_alias старый текст => новый текст
/route_tail source_alias target_alias маркер => новый блок
/route_clear source_alias target_alias
/route_include source_alias target_alias bmw,x5,m3
/route_exclude source_alias target_alias audi,mercedes
/route_filters_clear source_alias target_alias
/replay_last source_alias 10
```

Команды оставлены как запасной путь, но основной сценарий теперь кнопочный.

## Данные

Конфиг хранится в:

`data/config.json`

Там лежат:

- админы
- каналы
- маршруты
- кэш последних source-постов
- последний `update_id`
